## Core Thinking
- 为什么要有文档？**`文档是产品最核心的说明书`**, 它是**最完整**、**最系统**、**最符合人脑**的表述方式。
- 一个产品需要的最少说明书包括哪些？
  - `Introduction`, 基础介绍, 样例侧重于**开箱即用、简单易行**
  - `Tool Manuals`, 工具手册, 样例侧重于**系统完整、效率至上**
  - `Legislations`, 法律护盾, 样例侧重于**隐私合规、方法原则**
  - `Restrictions`, 适用禁忌, 样例侧重于**实验事实、安全责任**
- 如何写好产品说明书?
  - 使用好的Writing Tool, 例如`Markdown X`
  - 使用优秀的Style Tool, 例如`FumaDocs`
  - `使用系统的写作方法`
